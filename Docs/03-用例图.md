## 1 参与者定义

### 1.1 主要参与者 (Primary Actors)

| Actor ID | 名称 | 说明 | 对应用户类别 |
|---------|------|------|------------|
| **A1** | 业务分析师 (Analyst) | 平台核心用户，使用可视化界面构建和运行管道，无需技术背景 | 业务分析师 |
| **A2** | 数据工程师 (DataEngineer) | 高级用户，构建复杂管道，使用 API 集成，编写自定义 SQL/Python 转换 | 数据工程师 |
| **A3** | 系统管理员 (Admin) | 管理用户权限、系统配置、监控平台健康 | 系统管理员 |

### 1.2 次要参与者 / 外部系统 (Secondary Actors)

| Actor ID | 名称 | 说明 |
|---------|------|------|
| **S1** | SFTP 服务器 (SFTPServer) | 服务商放置 CSV/Excel 文件的 SFTP 服务器，系统通过 paramiko 连接并拉取文件 |
| **S2** | 邮件服务 (EmailService) | 发送告警和报告通知的 SMTP 服务 |
| **S3** | SSO/LDAP 服务 (AuthProvider) | 企业身份认证提供商（Phase 2 功能）|
| **S4** | Webhook 接收方 (WebhookTarget) | 接收执行状态通知的外部系统（Slack/Teams 等，Phase 2 功能）|
| **S5** | 调度引擎 (SchedulerEngine) | Prefect 3.x 定时调度系统（Phase 2 功能）|

## 5 关键流程序列图

### 5.1 创建并执行管道（端到端主流程）

```mermaid
sequenceDiagram
    actor 用户 as 业务分析师
    participant UI as Web前端
    participant API as REST API
    participant PE as 管道执行引擎
    participant DS as 数据源系统
    participant DT as 数据目标系统
    participant NS as 通知服务

    用户->>UI: 点击「新建管道」
    UI->>API: POST /api/v1/pipelines
    API-->>UI: 返回新管道 ID
    UI-->>用户: 打开设计器画布

    用户->>UI: 拖拽数据源节点并配置
    用户->>UI: 拖拽转换节点并配置规则
    用户->>UI: 拖拽目标节点并配置
    用户->>UI: 点击「预览」查看节点输出

    UI->>API: POST /api/v1/pipelines/{id}/preview
    API->>DS: 抽取样本数据 (LIMIT 100)
    DS-->>API: 返回样本数据
    API-->>UI: 返回预览结果
    UI-->>用户: 展示数据预览表格

    用户->>UI: 点击「发布管道」
    UI->>API: PUT /api/v1/pipelines/{id}/publish
    API-->>UI: 发布成功，版本号 v1
    UI-->>用户: 管道状态更新为「已发布」

    用户->>UI: 点击「立即执行」
    UI->>API: POST /api/v1/pipelines/{id}/runs
    API->>PE: 提交执行任务
    API-->>UI: 返回 Run ID
    UI-->>用户: 显示「执行中」状态

    loop 轮询执行状态（每 3 秒）
        UI->>API: GET /api/v1/runs/{run_id}
        API-->>UI: 返回当前状态和节点进度
    end

    PE->>DS: 抽取完整数据
    DS-->>PE: 数据流返回
    PE->>PE: 执行转换逻辑
    PE->>DT: 写入目标
    DT-->>PE: 写入成功确认

    PE->>API: 更新执行状态 → 成功
    API-->>UI: 状态：成功，处理 50,000 行，耗时 23s
    UI-->>用户: 显示执行成功 + 统计摘要
    API->>NS: 触发成功通知（如已配置）
    NS-->>用户: 发送邮件/Webhook 通知
```

### 5.2 数据源配置与连接测试

```mermaid
sequenceDiagram
    actor 用户 as 业务分析师
    participant UI as Web前端
    participant API as REST API
    participant CM as 连接管理器
    participant DS as 数据源系统

    用户->>UI: 打开「数据源管理」→「新建数据源」
    UI-->>用户: 显示数据源类型选择（卡片式）

    用户->>UI: 选择数据源类型（如 PostgreSQL）
    UI-->>用户: 显示该类型的配置表单

    用户->>UI: 填写连接参数（主机/端口/用户名/密码/数据库名）
    用户->>UI: 点击「测试连接」

    UI->>API: POST /api/v1/datasources/test（密码加密传输）
    API->>CM: 创建临时连接
    CM->>DS: 发起连接请求

    alt 连接成功
        DS-->>CM: 连接建立成功
        CM->>DS: 执行验证查询（SELECT 1）
        DS-->>CM: 查询成功
        CM-->>API: 返回成功 + 数据库版本信息
        API-->>UI: {success: true, dbVersion: "PostgreSQL 16.2"}
        UI-->>用户: 绿色提示「连接成功」，显示数据库版本

        用户->>UI: 点击「保存数据源」
        UI->>API: POST /api/v1/datasources（密码加密存储）
        API-->>UI: 返回数据源 ID
        UI-->>用户: 保存成功，跳转至数据源列表
    else 连接失败
        DS-->>CM: 连接被拒绝 / 超时 / 认证失败
        CM-->>API: 返回错误类型和详情
        API-->>UI: {success: false, errorCode: "AUTH_FAILED", message: "认证失败，请检查用户名和密码"}
        UI-->>用户: 红色提示具体错误原因
        Note over 用户: 修改参数后重新测试
    end
```

### 5.3 定时调度配置与失败告警

```mermaid
sequenceDiagram
    actor 用户 as 业务分析师
    participant UI as Web前端
    participant API as REST API
    participant SCH as 调度引擎
    participant PE as 管道执行引擎
    participant NS as 通知服务
    participant Email as 邮件服务

    用户->>UI: 进入管道调度配置页
    用户->>UI: 选择「每天 08:00 执行」
    用户->>UI: 配置失败通知收件人
    用户->>UI: 点击「启用调度」

    UI->>API: PUT /api/v1/pipelines/{id}/schedule
    API->>SCH: 注册 Cron 任务（0 8 * * *）
    SCH-->>API: 注册成功
    API-->>UI: 返回调度已激活，下次执行时间
    UI-->>用户: 显示下次执行时间

    Note over SCH: 次日 08:00 定时触发
    SCH->>API: POST /api/v1/pipelines/{id}/runs（系统触发）
    API->>PE: 提交执行任务

    alt 执行成功
        PE-->>API: 执行成功，处理 50,000 行
        API->>SCH: 记录成功，更新下次执行时间
    else 执行失败
        PE-->>API: 执行失败：目标数据库连接超时
        API->>NS: 触发失败告警事件
        NS->>Email: 渲染告警邮件（含错误摘要和日志链接）
        Email-->>用户: 收到告警邮件

        Note over API,SCH: 按重试策略自动重试（1/3）
        SCH->>PE: 30 分钟后重新触发
    end
```

### 5.4 用户登录与权限验证

```mermaid
sequenceDiagram
    actor 用户
    participant UI as Web前端
    participant API as REST API
    participant Auth as 认证服务
    participant DB as 元数据存储

    用户->>UI: 输入用户名和密码
    UI->>API: POST /api/v1/auth/login

    API->>DB: 查询用户信息
    DB-->>API: 返回用户记录（含 bcrypt 哈希密码和角色）

    API->>Auth: 验证密码（bcrypt compare）

    alt 验证成功
        Auth-->>API: 验证通过
        API->>Auth: 生成 JWT（含用户ID、角色、过期时间）
        API-->>UI: {token: "...", refreshToken: "...", role: "editor"}
        UI-->>用户: 跳转到主页，按角色渲染菜单
    else 验证失败
        Auth-->>API: 验证失败
        API-->>UI: {error: "用户名或密码错误"}
        UI-->>用户: 显示错误提示（不区分是账号还是密码错误）
    end

    Note over 用户,API: 后续 API 请求携带 JWT
    用户->>UI: 访问受权限保护的功能
    UI->>API: GET /api/v1/... (Authorization: Bearer {token})
    API->>Auth: 验证 JWT 签名和过期时间
    API->>Auth: 检查用户角色是否有权访问该资源

    alt 有权限
        Auth-->>API: 权限通过
        API-->>UI: 返回数据
    else 无权限
        Auth-->>API: 权限不足
        API-->>UI: 403 Forbidden
        UI-->>用户: 显示「无权限访问」提示
    end
```

## 6 用例规格说明

### UC-32 可视化设计管道

| 属性 | 内容 |
|------|------|
| **用例编号** | UC-32 |
| **用例名称** | 可视化设计管道 |
| **参与者** | A1 业务分析师（Primary）、A2 数据工程师（Primary） |
| **优先级** | Must Have |
| **关联需求** | FR-B-01、FR-B-03、FR-B-04 |

**前置条件**:
- 用户已登录系统（角色为 Editor 或 Admin）
- 至少存在一个已配置的数据源
- 管道已创建（处于草稿状态）

**后置条件**:
- 管道配置已持久化保存
- 管道可进入发布或执行流程

**主成功场景**:

| 步骤 | 用户操作 | 系统响应 |
|------|---------|---------|
| 1 | 进入管道设计器 | 展示空画布，左侧节点库，右侧隐藏配置面板 |
| 2 | 从左侧拖拽「数据源」节点到画布 | 节点出现在画布，右侧弹出配置面板 |
| 3 | 在配置面板选择已有数据源，配置查询 | 节点标题更新为数据源名称，连接端口激活 |
| 4 | 拖拽一个或多个「转换」节点到画布 | 节点按类型显示不同图标 |
| 5 | 从源节点输出端口拖线到转换节点输入端口 | 连线成功，箭头显示数据流方向 |
| 6 | 配置转换节点参数 | 实时校验配置合法性 |
| 7 | 拖拽「数据目标」节点并连线 | 管道结构形成完整 DAG |
| 8 | 系统实时验证 DAG 合法性 | 右上角显示「结构正常」绿色标识 |
| 9 | 点击「保存」 | 配置持久化，Toast 提示「已保存」 |

**扩展场景**:

- **4a. 用户希望预览节点输出**:
  1. 点击节点上的「预览」按钮
  2. 系统抽取样本数据并执行到该节点的转换（最多 100 行）
  3. 在底部面板展示数据表格预览

- **8a. 管道结构存在错误（节点未连接）**:
  1. 孤立节点高亮显示红色边框
  2. 右上角显示「结构异常」红色标识
  3. 悬停显示具体错误（如「该节点未连接到管道」）
  4. 用户修正错误，错误提示消失

**异常场景**:

- **3b. 数据源连接超时（预览时）**:
  1. 显示连接超时提示
  2. 提供「重试」按钮或「更换数据源」链接

**业务规则**:

| 规则 ID | 规则内容 |
|---------|---------|
| BR-01 | 管道有且仅有一个起始节点（数据源节点），且该节点无输入端口 |
| BR-02 | 管道至少有一个终止节点（数据目标节点），且该节点无输出端口 |
| BR-03 | 不允许存在孤立节点（未连接任何连线的节点） |
| BR-04 | 连线不能形成循环（DAG 约束） |
| BR-05 | 节点间连线方向只能从上游输出端口到下游输入端口 |

**界面要求**:
- 画布支持鼠标滚轮缩放（10%~200%），缩放中心跟随鼠标位置
- 支持多选节点（Ctrl+Click 或框选）批量移动
- 支持键盘快捷键：Ctrl+Z 撤销、Ctrl+Y 重做、Delete 删除选中节点
- 节点连线支持拖拽调整弯曲路径

sequenceDiagram

    用户->>UI: 进入「数据源管理」→ 选择 SFTP 数据源

    UI->>API: GET /api/v1/datasources/{id}/sftp/list

    SFTP-->>API: 文件内容（字节流）

    API->>DuckDB: read_csv(file_bytes, auto_detect=True)

### 7.3 Raw→DWD→DWS→ADS 全链路执行

```mermaid
sequenceDiagram
    actor 用户 as 业务分析师
    participant UI as Web 前端
    participant API as FastAPI
    participant DuckDB as DuckDB 处理引擎
    participant MySQL as MySQL

    用户->>UI: 点击「执行全链路」
    UI->>API: POST /api/v1/executions\n{datasource_id, layers: ["raw→dwd","dwd→dws","dws→ads"]}

    Note over API: 生成 Run ID，状态设为 running

    rect rgb(230, 245, 255)
        Note over API,MySQL: Step 1: Raw → DWD
        API->>MySQL: SELECT * FROM etl_raw.<表名>
        MySQL-->>API: 原始数据（分批读取）
        API->>DuckDB: 执行字段映射 SQL + 过滤条件
        DuckDB-->>API: 转换后数据
        API->>MySQL: INSERT INTO etl_dwd.<用户定义表名>
    end

    rect rgb(230, 255, 230)
        Note over API,MySQL: Step 2: DWD → DWS
        API->>MySQL: SELECT * FROM etl_dwd.<表名>
        MySQL-->>API: 明细数据
        API->>DuckDB: 执行 GROUP BY + 聚合函数 SQL
        DuckDB-->>API: 聚合结果
        API->>MySQL: INSERT INTO etl_dws.<用户定义表名>
    end

    rect rgb(255, 245, 220)
        Note over API,MySQL: Step 3: DWS → ADS
        API->>MySQL: SELECT * FROM etl_dws.<表名>
        MySQL-->>API: 汇总数据
        API->>DuckDB: 执行字段选取 + 排序 SQL
        DuckDB-->>API: 输出数据
        API->>MySQL: INSERT INTO etl_ads.<用户定义表名>
    end

    API->>MySQL: UPDATE executions SET status='success'
    API-->>UI: {run_id, status: "success", \nrows_raw: 1200, rows_dwd: 1100,\nrows_dws: 45, rows_ads: 45}
    UI-->>用户: ✅ 全链路执行完成，可查看各层数据或导出 ADS 结果
```
